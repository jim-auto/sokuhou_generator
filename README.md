# 即報ジェネレータ

入力フォームから即報文を自動生成する、1ページ完結の静的Webアプリです。HTML / CSS / JavaScriptのみで動作するため、ビルドなしでGitHub Pagesに公開できます。

## ファイル構成

```text
.
├── index.html
├── styles.css
├── script.js
└── README.md
```

## 使い方

1. `index.html` をブラウザで開きます。
2. `詳細入力` では左側のフォームに、エリア、日時、反応、結果、振り返りを入力します。
3. `一括メモ` ではメモをまとめて貼り付けると、ラベルやキーワードから項目を推定して生成します。
4. 右側に即報文の候補が3つリアルタイムで表示されます。
5. ジャンルは `スト師標準`、`パワギラ即`、`目ビーム`、`即確アポ`、`反省職人`、`淡々速報` から選べます。
6. 長さは `短め`、`標準`、`詳しめ` から選べます。
7. 結果タイプを選ぶと、`満即`、`準即`、`番ゲ`、`LINE交換`、`即確アポ` などに合わせて締めコメントが変わります。
8. 案件プロフィールの `出力に入れる` をオンにすると、年齢、職業、スト値、カップ、スペ値、JOJOを短く出力へ混ぜられます。
9. `ランダム` ボタンでシードを変えると、同じ入力でも言い回しや締めコメントが変わります。
10. 各候補の `コピー` ボタンで、その候補だけをクリップボードへコピーできます。

入力内容はブラウザの `localStorage` に自動保存されます。同じブラウザで再度開くと、前回の入力内容が復元されます。

## GitHub Pagesで公開する方法

1. このフォルダの内容をGitHubリポジトリへpushします。
2. GitHubのリポジトリ画面で `Settings` を開きます。
3. 左メニューの `Pages` を開きます。
4. `Build and deployment` の `Source` で `Deploy from a branch` を選びます。
5. `Branch` で `main` と `/ (root)` を選び、`Save` を押します。
6. 数十秒から数分後、表示されたGitHub PagesのURLからアクセスできます。

## カスタマイズ

出力テンプレートは `script.js` の以下の関数で分かれています。

- `buildShortReport`
- `buildStandardReport`
- `buildDetailedReport`

ジャンルごとの言い回しは `NANPA_GENRE_CONFIG` にまとめています。新しいジャンルを追加する場合は、HTMLのラジオボタンと `NANPA_GENRE_CONFIG` を追加してください。
結果タイプごとの締めコメントやタグは `RESULT_TYPE_CONFIG` にまとめています。

ナンパ師向けの出力は `TWEET_BUILDERS` に分けています。短い行、要点ラベル、締めコメント、ハッシュタグを調整したい場合はこの定義を編集してください。

ランダム生成はシード付きです。同じシードと同じ入力なら同じ生成結果になり、シードを変えるとラベル、締めコメント、ハッシュタグが変わります。
ジャンルごとのランダム候補には `即確アポ`、`パワギラ即`、`目ビーム` などの用語も入れています。
出力は最大140文字に制限され、`短め` は一文で生成されます。
生成候補は3つ表示され、各候補に文字数カウンターが付きます。
